依存句法分析

更新时间：2020-03-10



简介

通过分析语⾔单位内成分之间的依存关系揭⽰其句法结构。 直观来讲，依存句法分析识别句⼦中的“主谓宾”、“定状补”这些语法成分，并

分析各成分之间的关系。

  • 接口类型：通过 REST API 的方式提供的通用的 HTTP 接口。适用于任意操作系统，任意编程语言

  • 接口参数：具体详见请求说明。


调用流程

  1. 通过sophia AI开放注册进行调用。

  2. 返回识别结果：识别结果会即刻返回，采用 JSON 格式封装，如果调用成功，识别结果放在 JSON的“data”字段中，统一采用 utf-8 方式

    编码。具体详见返回说明。


请求说明

请求示例

HTTP 方法：POST

请求URL： http://106.13.122.156:8088/v1/api/zhujian/participle


请求参数

参数 是否必选 类型 可选值范围 说明
queries 是 list - 待分析⽂本

flags

是 string - 功能模块名称：分词和词性标注
locale 是 string - 该参数默认可不加，默认为简体，如需要繁体，则增加此参数 zh_TW


请求示例 

{“queries”:[”我喜欢电影”]
“flags”:”dependency”
}


返回说明


  1. Head 参数说明：

字段 说明 值／remark／消息
stateCode 状态码

0表示成功：成功

1000 在 flag 参数中有语法错误: 空 flag 或传入错误的 flag 参数

1001 表示错误的命令请求格式： JSON 格式错误

2000 表示请求的操作超时终止：模块处理超时

2001 表示未执行请求的操作：⽂本超过指定长度（TBD）

2002 表示未执行请求的操作：请求超过指定数量


cost 使用时间 毫秒（float）


  1. body 参数说明：


字段 类型 说明
query string 待分析文本
dependency string 依存句法分词结果
